# Repository Audit — May 2026

**Scope:** porschedigital-technology-radar monorepo (pnpm workspace, packages
`techradar` + `create-techradar`).
**Date:** 2026-05-07.
**Mandate:** Analysis only. No code changes. Concrete proposals and
optimization potentials, ranked by impact.
**Verdict:** No correctness bugs found. All findings are quality, drift-
prevention, and surface-area improvements. The repo is highly disciplined
(5-arm steering harness, 32 ADRs, per-directory AGENTS.md hierarchy,
OpenSSF Scorecard, OIDC publishing).

---

## 0. Audit Checklist (15 dimensions)

The audit ran against these dimensions:

1. Documentation ↔ code drift (root + package AGENTS.md, READMEs, HARNESS.md)
2. AGENTS.md per-directory coverage and freshness
3. ADR coverage, ordering, and references from AGENTS.md
4. Harness completeness (lint / typecheck / test / arch / sec / quality / a11y / build)
5. Code anti-patterns (type escapes, console use, dangerouslySetInnerHTML, oversized files)
6. Scaffolder ↔ framework alignment (`create-techradar` vs `techradar`)
7. Dependency hygiene (Dependabot, license scan, lockfile, SHA-pinning)
8. Test quality and CI parity (`pnpm test` on a fresh clone)
9. Skills surface and AGENTS.md token cost
10. CI workflows + Husky hook parity with documented "Definition of Done"
11. Workspace ownership (root-only vs package-delegated scripts)
12. Build pipeline (`build:data`, `build:themes`, OG images, static export)
13. Generated-output hygiene (gitignored artifacts, fresh-clone bootstrap)
14. Hidden coupling (rename shims, version-shaped strings, drift hazards)
15. Markdown and ADR consistency (numbering, cross-references, tone)

## 4. Tier 4 — Documentation Redundancy and Token Cost

- `packages/techradar/AGENTS.md` is **348 lines (~32 KB)**. It is the single
  largest doc agents will read for any package change.
- `docs/HARNESS.md` is **320 lines (~27 KB)** and overlaps with the AGENTS.md
  harness section.
- `packages/techradar/README.md` repeats command tables.

**Proposal.** Promote HARNESS.md as the *single* authoritative harness
document. Replace the harness section in `packages/techradar/AGENTS.md`
with a 6–10 line summary + link. Estimated reduction: **40–60 % of the
techradar AGENTS.md** without information loss.

**Structural target for `packages/techradar/AGENTS.md`:**

1. Stack & layout (10 lines)
2. Hot paths & generated artifacts (10 lines)
3. Definition of Done — link to HARNESS.md
4. Hard rules (no `as any`, no `dangerouslySetInnerHTML` outside SafeHtml,
   no `Next/Image`, etc.) — bullet list, no rationale
5. Pitfalls (the existing list)
6. ADR index (one-line per ADR, linked)

Total target: **120–150 lines.**

## 10. Tier 10 — Skill Optimization

The repo currently advertises only `dev-browser` via the runtime env.
For this codebase, the *useful* skill set is small:

- **`git-master`** — for any commit/rebase work touching ADRs or
  release-please config.
- **`ai-slop-remover`** — for trimming AGENTS.md / HARNESS.md per Tier 4.
- **`playwright`** or **`dev-browser`** — for verifying the static export
  visually after `check:build`.
- **`frontend-ui-ux`** — for SpotlightSearch refactor (C4) or any
  component work.

Recommend adding a "Recommended skills per task" table to root
`AGENTS.md` (≤ 10 lines).
